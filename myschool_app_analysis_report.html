All confirmed issues ranked by severity
P0 CRASH
ProGuard strips MySchoolApplication class
Fatal crash on Jun 7 logcat. Release APK fails immediately at startup. Add -keep class com.kartik.myschool.** { *; } or specific keep rule in proguard-rules.pro.
P0 DATA
Static AppCache causes stale/cross-user data
AppCache is a raw static class with no lifecycle awareness. On process restart or second login, stale data from previous session is served to UI — teachers can see each other's class data. Should be ViewModel-scoped or cleared on logout.
P0 LOGIC
Semester detection relies on string heuristics
In getMarksForStudentAndSemester() — falls back to checking if semesterId "contains '2'" or "contains 'second'" to determine semester number. Fragile. Will misroute marks if ID happens to contain "2".
P1 UX
HomeActivity has 700+ lines of inline logic
All navigation, toolbar logic, sidebar setup, offline detection, locale switching crammed into one Activity. God-object antipattern. Increases crash surface and makes debugging navigation bugs extremely hard.
P1 UX
No onboarding / empty state flow
New teachers see a blank HomeActivity with no guidance on "what to do first" (create school → add year → add class → add students). Top apps like ClassDojo show a step-by-step setup wizard. This is the #1 reason new users churn.
P1 PERF
Marks entry: full class Firestore fetch on every open
Even with TTL cache, the 5-min window means large classes (40+ students) must wait for Firestore on cold open. Switching students causes visible lag. No pagination or lazy load in the marks grid.
P1 DATA
No Firestore composite index — sorting done in Java
Students sorted in-memory after fetching all documents. With 200+ students this is O(N log N) on main thread. Should use Firestore orderBy with composite index (classId + rollNo).
P1 CRASH
NetworkCallback registered/unregistered in onStart/onStop (not safe)
If connectivityManager is null or networkCallback not yet initialized when onStop runs, silent exception is swallowed with e.printStackTrace() — not reported to Crashlytics. Should null-check and use try-finally.
P2 UX
No undo/confirmation on destructive actions
Deleting a student, deleting marks, promoting students — no snackbar undo. Firestore delete is permanent. ClassDojo and Google Classroom both offer 30-second undo on destructive actions.
P2 A11Y
Missing content descriptions on icon-only buttons
Toolbar menu/help/calc buttons set contentDescription only in setupDrawerActions — many icon-only buttons elsewhere have none. TalkBack users cannot navigate the app.
P3 NICE
No tablet / large-screen layout
All layouts are phone-portrait only. Google Play now penalises apps without large-screen support in rankings. A two-pane layout (class list + student detail) would dramatically improve tablet UX.
P3 NICE
Zero automated tests
androidTest and test directories exist but are empty. GradeCalculator, PdfLocalizer, and mergeRecords() logic are critical paths that should have unit tests. Any mark calculation regression goes undetected.
